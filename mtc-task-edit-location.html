General Informations
Frequency
Time Based Inspection based on time (date).
Usage Based Inspection by need (in hours, km, etc.).
Cycle Based Inspection based on cycle.

Set the schedule for your task using a cron expression.

“Every day”

Example:
If you want to repeat “On the first day of every month”, then:
  • Fill the day (Month) column with: 1
  • Fill the month column with: *
  • Fill the Day (Week) column with : *

Symbol Guide

*
any value
,
value list separator
-
range of values
/
step values

Start On
  • This is the first date MTC is due.
  • It’s used when the task happens only once (e.g., initial setup inspection).

Example:
Inspection starts on Jan 1, 2025 → next due = Jan 1, 2025

Repeat Every
  • This is used when you want MTC to happen repeatedly.
  • Based on the last inspection date or "Start On" date if it’s the first one.

Example:
Repeat every 90 days → Jan 1, Apr 1, Jul 1...

First-Time On + Repeat Every
  • Use this if you want the first MTC to start on a specific date, and then repeat regularly.
  • It will calculate all next dates based on that first date + repeat.

Example:
First Time On: Feb 15, 2025
Repeat Every: 6 months → next: Aug 15, 2025, then Feb 15, 2026...

Unit
  • For time based, select one of day, week, or month
  • For usage based, input the unit according to the maintenance task. For example, oil change is measured in KM.
  • For cycle based, it is definitely with cycle
Repeat

Purpose: To execute repetitive tasks

Day(Month)
  • Numeric input, range from 1 to 31
  • By default it is filled with * which means "Every"
Month(s)
  • Numeric input, range from 1 (January) to 12 (December)
  • Example: 1, 4, 12 → Triggers in January, April, and December
  • By default it is filled with * which means "Every"
Day(Week)
  • Numeric input, range from 1 (Monday ) to 7 (Sunday)
  • By default it is filled with * which means "Every"
One Time

Purpose: To execute tasks on specific date
